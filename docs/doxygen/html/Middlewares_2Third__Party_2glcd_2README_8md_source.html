17 Introduction
18 ============
19 
20 Welcome to GLCD, an open source graphic LCD library written by Andy Gock.
21 
22 Author's web site: http://agock.com/
23 
24 GitHub repository: https://github.com/andygock/glcd
25 
26 This library has been written cleanly, to allow easy modification for use with different microcontroller devices and controller chipsets. Logic relating to devices and controllers are palced in seperate files and specific implementations can be chosen by the use of special defined symbols.
27 
28 It is suitable for monochrome (black and white) LCDs with page by page data and command write style data transfer protocol. It is not suitable for color graphic LCDs.
29 
30 Supported controllers and chipsets
31 ----------------------------------
32 
33 Works with:
34 
35 - PCD8544 based LCDs, e.g Nokia 3110 and 5110 LCDs
36 - ST7565R / ST7565P serial interface
37 - NT75451 parallel interface (used on NGX BlueBoards)
38 
39 The following Newhaven displays have been physically tested with and confirmed working:
40 
41 - NHD-C12864WC-FSW-FBW-3V3-M
42 - NHD-C12864A1Z-FSW-FBW-HTT
43 - NHD-C12832A1Z-FSW-FBW-3V3
44 - ZOLEN-12864-FFSSWE-NAA
45 
46 Supported microcontrollers
47 --------------------------
48 
49 MCUs supported:
50 
51 - Atmel AVR 8-bit
52 - NXP LPC111x ARM Cortex-M0
53 - NXP LPC11Uxx ARM Cortex-M0
54 - ST STM32 F0 ARM Cortex-M0
55 - ST STM32 F4 ARM Cortex-M4
56 - Microchip PIC24H (and probably other 16-bit MCUs)
57 
58 Development boards tested on (with on-board LCD):
59 
60 - NGX BlueBoard LPC11U37 (with on-board NT75451 graphic LCD)
61 
62 Development boards tested on (without on-board LCD):
63 
64 - Microstick II with PIC24H and Nokia 3310/5110 LCD, ST7565R and ST7565P
65 - ST Nucleo F401RE.
66 
67 ### Special note
68 
69 Not all combinations of microcontroller platform and LCD controllers are supported out of the box. However you can edit the files `devices/` and `controllers/` and add your desired combination. More information on how to do this can be read in the doxygen documentation.
70 
71 Setup of symbols for compiler
72 -----------------------------
73 
74 The following symbols need to be defined for the compiler:
75 
76 Pick microcontroller type (pick one only):
77 
78  GLCD_DEVICE_LPX111X
79  GLCD_DEVICE_LPX11UXX
80  GLCD_DEVICE_AVR8
81  GLCD_DEVICE_STM32F0XX
82  GLCD_DEVICE_STM32F4XX
83  GLCD_DEVICE_PIC24H
84 
85 Pick LCD controller type (pick one only):
86 
87  GLCD_CONTROLLER_PCD8544
88  GLCD_CONTROLLER_ST7565R
89  GLCD_CONTROLLER_NT75451
90 
91 For ST7565P controllers, treat as ST7565R. For most if not all parts here, they behave the same way.
92 
93 If using a parallel interface LCD (e.g NT75451 on NGX BlueBoard):
94 
95  GLCD_USE_PARALLEL
96 
97 When using SPI controllers:
98 
99  GLCD_USE_SPI
100 
101 Note the SPI symbol isn't actually checked by the source at the moment, and it is fine if it is not used. It is for forward compatibility only. One day I may decide to check for it.
102 
103 For the Newhaven displays using ST7565 based controllers listed above which have been tested as working, there
104 are certain initialisation sequences which should be followed, and this may vary
105 from display to display. To force a certain (and tested) initialisation
106 sequence, define one of the following:
107 
108  GLCD_INIT_NHD_C12832A1Z_FSW_FBW_3V3
109  GLCD_INIT_NHD_C12864A1Z_FSW_FBW_HTT
110  GLCD_INIT_NHD_C12864WC_FSW_FBW_3V3_M
111  GLCD_INIT_ZOLEN_12864_FFSSWE_NAA
112 
113 If you don't specify a NHD model, ST7565 controller selection will default to `GLCD_INIT_NHD_C12864WC_FSW_FBW_3V3_M` sequence.
114 This however may change in the future.
115 
116 To set a reset time, used by the `glcd_reset()` function, set `GLCD_RESET_TIME` to desired duration in milliseconds.
117 
118 These symbols need to be set in the configuration options of your IDE, usually
119 in the "defined symbols" section, or they can be defined in a makefile
120 as `-D` options.
121 
122 Example:
123 
124  -DGLCD_DEVICE_LPC111X
125 
126 Delay Timing
127 ------------
128 
129 Some operations such as sending a reset pulse, requires the use of a delay timer. The library will refer to a
130 external function called `delay_ms(t)` where t is the delay required in milliseconds. Please ensure you have
131 this function elsewhere in your program.
132 
133 If you are using avr-gcc with Atmel devices, you can force the library to use the built-in `_delay_ms()` function
134 by setting the compiler symbols:
135 
136  GLCD_USE_AVR_DELAY
137  __DELAY_BACKWARD_COMPATIBLE__
